14-RecyclerView

RecyclerView
- 리스트뷰의 개선판
- 리스트뷰의 단점
    - 매번 새롭게 뷰를 만들어줘야 한다
    - 뷰홀더를 사용하더라도 여전히 성능 이슈가 있다
        -> 리사이클러뷰는 뷰홀더가 기본적으로 사용된다
    - 유연하지 않다
            - 간단한 변경으로 여러가지 기능 구현이 불가능하다
            - 수정이 용이하지 않다

- 장점
    - 유연하다
        - 세로, 가로, 그리드, 스태거드 그리거드 방향으로 아이템 배치 가능
        - 아이템이 들어갈 뷰를 동적으로 변경하는것이 가능하다

- 단점아닌 단점
    - 사용자의 스크롤이 속도가 재활용 되는 속도보다 빠른 경우 로딩이 걸린다
        -> 그러면 선택사항 아닌가?
            - 로딩을 보여주기 싫다 -> AddView
            - 최적화가 더 중요하다 -> RecyclerView
        - (X) AddView보다 RecyclerView가 훨씬 우월하다 + (사실 이런경우가 거의 없음)
            -> 그럼 앱에서 스크롤할때 발생하는 로딩은 뭐냐?
                - 아직 뒤에 그릴 뷰의 데이터를 서버로부터 받지 못했기 때문(인터넷 시간)
